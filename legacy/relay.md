# 次日接力方案解读（relay）

本文档用于解释 UI 里的“次日接力方案（排除一字板）”怎么读、每个字段含义、过滤原因代表什么。

## 1) 功能定位

- 这是**次日交易决策辅助**，不是回测展示页。
- UI 仅读取数据库里的 `model_relay_pool` 结果，不在页面里临时训练模型。
- “是否入选=是”表示：在当前阈值和过滤规则下，当天候选里优先级最高、可执行的一只。

## 2) 数据来源与更新链路

- 日线行情：`stock_daily`
- 复盘池：`daily_review_akshare_pool`（现已由 Xuangubao/涨停宝接口写入，表名保留兼容）
- 涨停明细：`daily_review_xgb_limit_up`
- 接力打分结果：`model_relay_pool`

推荐更新顺序：

1. `everyday.py`（基础行情/主模型）
2. `everydayReview.py`（review池 + relay打分）

## 3) 页面由两部分构成

### A. 顶部摘要（relay summary）

主要字段解读：

- `明日市场`：市场阶段（强/中/弱）文字结论
- `仓位建议`：对应阶段给出的仓位建议
- `聚焦`：建议关注方向
- `涨停/跌停/炸板率/最高连板`：当日市场情绪核心指标
- `阈值`：当前入选阈值（可由 UI 输入框覆盖）
- `模型`：本次结果对应的模型版本
- `过滤规则`：当前执行约束（TopK、连板上限、跳空区间、风险档）
- `入选`：最终被选中的股票代码（若无则为空）
- `原因`：入选结果或未入选原因说明

### B. 下方表格（grid）

列含义：

- `代码/名称`：股票标识
- `评分`：用于排序和入选判定的模型分值（`model_score`）
- `原始概率`：模型原始概率（`model_prob`）
- `连板`：候选当日连板高度（`board_count`）
- `当日涨跌`：信号日涨跌（`ret1`）
- `是否入选`：本日是否被策略最终选中
- `过滤原因`：未入选时的主要拦截原因
- `开盘预案/买点触发/不及预期/止盈离场`：执行建议（交易提示，不是硬过滤）

## 4) 入选逻辑（按顺序）

候选是按 `rank_no` / `model_score` 排序后处理，流程如下：

1. 仅看前 `TopK` 个候选
2. `评分 >= 阈值`（阈值来自 UI 输入或模型默认值）
3. 连板过滤：`board_count <= max_board_filter`（若配置了）
4. 风险过滤（若配置了）：
   - `broken_rate <= max_broken_rate_filter`
   - `red_rate >= min_red_rate_filter`
   - `limit_down_count <= max_limit_down_filter`
   - `pullback <= max_pullback_filter`
5. 可交易过滤（规则 + 次日开盘）：
   - 必须有 `T+1` 交易日和开盘价
   - 若设置跳空窗口：`gap = next_open / close - 1` 需在 `[gap_min, gap_max]`
   - 若启用“涨停不可买”：T+1 开盘若接近涨停按不可成交处理
6. 第一个完全通过的候选即“是否入选=是”

## 5) 过滤原因说明

UI 文案与触发条件对应如下：

- `通过过滤`（`ok`）：通过全部条件
- `分位分数低于阈值`（`threshold_blocked`）：评分未达阈值
- `连板高度超限`（`board_blocked`）：连板超过上限
- `风险过滤未通过`（`risk_blocked`）：触发风险约束
- `次日开盘跳空超限`（`gap_blocked`）：开盘跳空不在允许区间
- `次日一字/涨停按规则不可买`（`rule_blocked`）：规则判定不可成交
- `缺失次日开盘价/交易日`（`bad_buy_quote` 等）：数据不完整无法执行

细分原因（常见）：

- `gap_below_min`：开盘跳空低于下限
- `gap_above_max`：开盘跳空高于上限
- `limit_up_unfilled`：开盘已接近涨停，按规则视为买不到
- `broken_rate_high`：炸板率过高
- `red_rate_low`：红盘率过低
- `limit_down_high`：跌停家数过高
- `pullback_high`：日内回撤过大

## 6) “开盘预案”等执行字段如何解读

这些字段是根据“评分相对阈值溢价 + 连板高度”生成的交易提示：

- 高连板（如 4 板及以上）：更强调回封确认和谨慎追高
- 溢价高（评分明显高于阈值）：可更积极，但仍需分时确认
- 溢价低：偏观察、少试错

注意：这些字段用于盘中执行参考，不参与“是否入选”的硬判定。

## 7) 建议的阅读顺序

1. 先看摘要里的`明日市场/仓位建议`
2. 再看`阈值 + 过滤规则`是否符合你的风险偏好
3. 看表格中`是否入选=是`的那只
4. 对照`过滤原因`理解其他票为何被淘汰
5. 盘中按`开盘预案/买点触发/不及预期/止盈离场`执行

## 8) 常见问题

- 表格空：
  - 先确认是否已跑 `everydayReview.py`
  - 再确认对应交易日在 `model_relay_pool` 有数据
- 全部都“未入选”：
  - 常见是阈值过高，或风险/跳空规则过严
- 想手动更激进/保守：
  - 在 UI 调整“接力阈值(分位)”（0.50 ~ 0.99）
